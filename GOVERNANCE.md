# Governance

Open Scrapers Toolkit is intended to grow into a community-maintained library of public scrapers.

## Maintainer goals

- keep the project easy to fork and reuse
- keep scraper behavior transparent
- review source legality and ethics before merging
- favor simple interfaces over framework-heavy abstractions

## Decision making

- Small changes can be merged after one maintainer review.
- New source integrations should include docs and a short explanation of why the source is appropriate.
- Breaking changes should be documented in the README and the relevant docs page.

## Roadmap direction

The project should grow across these areas:

- more official government and NGO feeds
- more academic and research indexes
- export options such as CSV and NDJSON
- scheduling and deployment support
- automated smoke tests for source health

## Community expectations

- Be respectful and constructive.
- Assume good intent, but prioritize source legality and user safety.
- If a source becomes questionable, the maintainers may remove it even if it is technically interesting.
