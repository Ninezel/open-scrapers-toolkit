# Security Policy

## Reporting a vulnerability

If you find a security problem in this project:

- Do not open a public issue with exploit details.
- Share the problem, affected files, and reproduction steps privately with the maintainers.
- Include the package manager output if the issue involves a dependency.

## Scope

Relevant issues include:

- command injection
- path traversal
- unsafe file writes
- accidental secret exposure
- dependency vulnerabilities with practical impact

## What to expect

- Maintainers should acknowledge a report quickly.
- We will confirm impact, prepare a fix, and credit reporters when appropriate.
- If the issue only affects a third-party source endpoint, we will document the mitigation or remove the scraper if needed.
